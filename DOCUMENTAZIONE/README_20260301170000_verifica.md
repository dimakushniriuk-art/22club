## 2) Query di verifica (Supabase SQL Editor)

### Conteggi ruoli (solo 4 ammessi)

```sql
SELECT role, COUNT(*) AS n
FROM public.profiles
GROUP BY role
ORDER BY role;
```

Atteso: solo righe con role IN ('admin','athlete','marketing','trainer').

### Constraint profiles.role

```sql
SELECT conname, pg_get_constraintdef(oid) AS def
FROM pg_constraint
WHERE conrelid = 'public.profiles'::regclass
  AND conname = 'profiles_role_check';
```

Atteso: `CHECK ((role = ANY (ARRAY['admin'::text, 'trainer'::text, 'athlete'::text, 'marketing'::text])))` (o equivalente).

### Policy senza ruoli legacy

```sql
SELECT policyname, tablename,
  LEFT(qual::text, 120) AS qual_preview
FROM pg_policies
WHERE schemaname = 'public'
  AND (
    qual::text LIKE '%''pt''%'
    OR qual::text LIKE '%nutrizionista%'
    OR qual::text LIKE '%massaggiatore%'
    OR qual::text LIKE '%''staff''%'
    OR qual::text LIKE '%''owner''%'
  );
```

Atteso: 0 righe. (Se compare ancora 'atleta' in qualche policy, è ammesso temporaneamente come valore per filtri athlete.)

### Policy che usano ancora pt_atleti

```sql
SELECT policyname, tablename
FROM pg_policies
WHERE schemaname = 'public'
  AND qual::text LIKE '%pt_atleti%';
```

Atteso: 0 righe (progress_photos e chat usano athlete_trainer_assignments).
